# המלצות לשיפור מבנה תוכן

## עמוד הבית (`/`)

**בעיה במבנה הקיים:**
- שני אלמנטי H1 ב-DOM (אחד למובייל, אחד לדסקטופ) - מנועי חיפוש רואים כפל
- VisionToExecutionSection: כרטיסים עם caption בלבד, ללא H2

**מה לשנות בכותרות:**
- H1: להשאיר אחד בלבד (להסיר כפילות)
- אין צורך לשנות את תוכן ה-H1 - "פיתוח אתרים, מערכות AI ושיווק שמייצר תוצאות!" כבר טוב

**כותרות חדשות מומלצות:**
- VisionToExecutionSection: להוסיף H2 "מה אנחנו בונים" מעל הכרטיסים
- כל כרטיס: caption יכול להישאר, או להפוך ל-H3 אם רוצים היררכיה עמוקה יותר

**מבנה מומלץ:**
```
H1: פיתוח אתרים, מערכות AI ושיווק שמייצר תוצאות!
H2: אפיון, פיתוח, עיצוב ושיווק מבוסס AI שמייצר צמיחה
  [כרטיסים]
H2: שאלות נפוצות
```

      }
    },
    {
      "@type": "Question",
      "name": "אילו שירותים VOW מציעה?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "עיצוב ומיתוג, פיתוח אתרים ב-AI, פיתוח מערכות, שיווק דיגיטלי, חשבונית דיגיטלית, רואה חשבון AI."
      }
    },
    {
      "@type": "Question",

        "@type": "Answer",

**הטמעה:** קומפוננטת `JsonLd` ב-`page.tsx` או ב-layout, עם מיפוי מ-FAQ_ITEMS.

## Layout ראשי (כל האתר)

**סוג Schema:** Organization

**למה מתאים:** מזהה את המותג ואת פרטי הקשר לכל האתר. `sameAs` מחזק Entity SEO - מקשר את VOW לפרופילים ברשתות.

**קוד JSON-LD:**

```json
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "VOW",
  "url": "https://uxellent.com",
  "logo": "https://uxellent.com/logo.png",
  "sameAs": [
    "https://www.linkedin.com/company/vow",
    "https://www.facebook.com/profile.php?id=61587713103366",
    "https://www.instagram.com/vow.for.success",
    "https://x.com/Vowsuccess"
  ],
  "contactPoint": {
    "@type": "ContactPoint",
    "email": "support@uxellent.com",
    "contactType": "customer service",
    "availableLanguage": ["Hebrew", "English"]
  }
}
```

**הערה:** עדכנו את `sameAs` לפי הפרופילים האמיתיים. LinkedIn - אם קיים, הוסף קישור מדויק.

**הטמעה:** ב-`layout.tsx` הראשי.
